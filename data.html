Building the Data

We started with the use of Python and an API to pull recipe data.

In python defintions were made to search groups of 100 recipes and then parse them into one JSON file per recipe. This method creates robustness to ensure future searches resulting in a known recipe would not create duplicates.

After a new recipe is introduced to the RawData folder, a definition is run to combine all the recipes into a single file to be consumed by the HTML/Javascript

Our Models

Cindy created one model per Carbs, Cholesterol, Fat, and Sugar. Unhealty was defined as Carbs >15, Cholesterol >300, Fat >77, Sugar >25

Machine Learning

The goal of this project was to run some machine learning to teach models to idenitfy recipes that were pulled that would be considered Healthy vs. Unhealthy

Each model used a recipe dataset identified by the API.

Regression Models: Models were generated and then dupmed to a pckl file and loaded later to predict many recipes

Neural Network Models: A model was also generated using tensorflow to create an neural network
